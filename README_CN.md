ComfyUI-TrucyNodes 节点详细使用手册

本节点包旨在解决 AI 影视分镜创作中资产溶图难、多模态模型性能分配不均、以及后期影视分辨率适配等实际痛点。

1\. TrucyKleinEncode (专业视觉编码节点)

这是针对 Flux2-Klein 模型设计的核心编码节点，负责将文字指令与最多 10 张图像资产（img1-img10）进行深度融合。

核心参数定义：

width / height: 设置最终生成的画面分辨率。默认 1920x1088。

main\_prompt\_ratio: 整体权重平衡系数。

数值为 0.5 时：文字指令与图片资产的影响力达到 1:1 平衡。

数值 大于 0.5：增强文字指令的权力（例如让动作更显著）。

数值 小于 0.5：增强图片资产的还原度（例如让人脸更像原图）。

non\_base\_alignment (对齐模式开关): \[关键选项]

Follow Node (W/H)：此模式下，RSA (Reference Square Area) 数值将被程序完全忽略。所有注入的图片都会被放置在与全局 width/height 相同的全黑画布上。这保证了所有资产与主画面拥有 1:1 的物理坐标对应关系，画质最高，但采样速度较慢。

Use RSA Scaling：在此模式下，RSA 数值被激活。只有在 base\_target 里选中的那张图（通常是场景图）会保持全画幅对齐；其他所有图片（人物、道具等）都会按照 rsa\_value 设定的面积进行等比缩小。这能大幅减轻显存压力，提升采样速度。

rsa\_value (Reference Square Area - 参考图面积基准):

仅在 Use RSA Scaling 模式下生效。它定义了非基准图在转换成特征时的像素总量。填入 1024 代表按 1024x1024 面积缩放。数值越高细节越好，数值越低速度越快。

base\_target:

决定将哪一个接口注入的图片作为“基准底图”输出给采样器（KSampler）。

如果你在 img5 连入了场景资产，这里请选 img5。

img1 - img10: 10 个独立的图像资产输入接口。

img1\_strength - img10\_strength: 对应每张图片的独立权重。拉到 0 代表彻底关闭该插槽，不消耗任何算力。

base\_mask: 遮罩输入。该遮罩将自动绑定在你选定的 base\_target 图片上。

提示词书写规则：

必须使用 img1 到 img10 标签。 代码内部已将输入接口与标签强制绑定。

正确示例：img1 的青年男子拿着 img2 的拐杖，站在 img5 的草地上。

2\. TrucyImageAdapter (图像分辨率适配节点)

由于 Flux 模型通常要求分辨率能被 16 整除（如 1088p），但影视流程标准是 1080p，此节点负责精准转换。

mode (模式选择):

Crop (Center): 居中裁切。这是 1088p 转 1080p 的最标准做法。它会自动从图片顶部切除 4 像素，底部切除 4 像素，保持画面 1:1 像素不变形。

Stretch: 拉伸。无视原图比例，强行压缩/拉伸到你设定的 target 尺寸。

target\_width / target\_height: 设置你最终想要的输出尺寸，默认 1920x1080。

3\. TrucyExcelReader (智能表格读取节点)

从 Excel 中读取脚本内容，并能自动处理文本中夹杂的参数。

excel\_path: 支持 Windows 资源管理器直接右键复制的带双引号 "" 路径。

row / column: 指定读取的行和列。

输出类型:

string: 单元格内的原始文本。

int / float: 节点会通过正则表达式提取单元格内第一次出现的数字。例如单元格为 "人物1.25在54号位置"，它会准确输出 float: 1.25, int: 1。
