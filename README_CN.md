# 🎬 ComfyUI-TrucyNodes (翠西节点包)
**专为 AI 影视工业化、分镜创作及视觉一致性设计的进阶套件**

---

## 📖 简介
本节点包旨在解决 AI 影视创作中多资产溶图难、构图不可控、以及多模态模型（如 **Flux2-Klein**）在采样过程中算力分配不均等核心痛点。它提供了一套从脚本读取到影视级分辨率适配的完整闭环工具。

## 🚀 核心节点详解

### 1. TrucyKleinEncode (影视分镜导演节点)
这是套件的核心，负责将文字指令与最多 10 张图像资产（img1-img10）进行深度融合。

#### 🛠️ 关键参数说明：
*   **`width` & `height`** 📏:
    *   设置画布的全局分辨率。
    *   *提示*：Flux2 系列模型建议设为 **1088** 以满足 16 字节对齐，防止报错。
*   **`main_prompt_ratio`** ⚖️:
    *   **权重天平**：控制提示词指令与参考图特征的博弈。
    *   **0.5 (缺省值)**: 完美 1:1 平衡，等同于最强溶图模式。
    *   **> 0.5**: 增强提示词权力（当模型不听从“打斗”、“奔跑”等动作指令时调高）。
    *   **< 0.5**: 增强资产还原度（当生成的角色长得不像参考图时调低）。
*   **`non_base_alignment` (对齐模式大开关)** 🔄:
    *   **`Follow Node (W/H)`**: 🚀 **极致画质模式**。
        *   **逻辑**：所有注入的资产图（img1-10）都会被强制放置在与全局 width/height 相同的全黑画布上进行编码。
        *   **性能**：此模式下 **RSA (Reference Square Area)** 的数值会被程序**完全忽略**。
        *   **优点**：资产与主画面拥有 1:1 的物理坐标对应关系，溶图精度最高，画质最稳。
    *   **`Use RSA Scaling`**: ⚡ **性能加速模式**。
        *   **逻辑**：仅在 `base_target` 中选中的底图保持全画幅，其他资产按 RSA 面积缩放。
        *   **性能**：**RSA 数值被激活**。大幅降低 KSampler 的计算压力。
*   **`rsa_value` (Reference Square Area - 参考图面积基准)** 📐:
    *   **生效条件**：仅在 `non_base_alignment` 设为 `Use RSA Scaling` 时起作用。
    *   **功能**：定义辅助资产（如人物、道具）在转换成特征时的像素总量。
    *   **推荐值**：1024。降低此值可极大提升采样速度；提高此值可增强资产纹理质感。
*   **`base_target`** 🎯:
    *   **逻辑**：指定哪张图是“空间基准”。被选中的图（通常选场景图 img5）将作为 KSampler 的物理起始底图。
    *   **注意**：`base_mask` (遮罩) 只会作用于此处选中的图片。
*   **`img1` - `img10` 接口** 🖼️:
    *   **标签绑定**：强制命名匹配。提示词中使用 `img1` 标签将精准调用 `img1` 接口的视觉特征。
    *   **强度控制**：各插槽自带独立强度滑块，设为 **0.0** 可物理切断该接口，不占显存。

### 2. TrucyImageAdapter (影视分辨率适配器)
解决从 Flux 的 **1088p** 到影视标准 **1080p** 的最后 8 像素适配问题。

*   **模式选项**:
    *   **`Crop (Center)`** ✂️: **[推荐]** 居中无损裁切。自动切除顶部 4 像素和底部 4 像素。保持画面 1:1 像素映射，绝无拉伸变形。
    *   **`Stretch`** 🎨: 强行挤压成目标尺寸，适用于不计较比例的快速适配。

---

### 3. TrucyExcelReader (智能脚本读取器)
实现分镜资产路径或提示词的自动化批量加载。

*   **路径清洗** 📂: 完美兼容从 Windows 资源管理器直接“右键复制路径”得到的带双引号 `""` 路径。
*   **数字嗅探** 🔢: 内置正则逻辑，自动提取文本中出现的**第一个**数字。
    *   *示例*：读取 "镜头05.5-最终版" -> 自动输出 float: **5.5**, int: **5**。

---

### 4. Audio同步工具组
*   **`Audio Detector & Padder`** ⏱️: 根据 FPS 计算音频精准帧数。支持“步长补齐”（如 250ms），确保 AI 视频时长永远对齐节奏，防止后期音画不同步。
*   **`Empty Audio Generator`** 🔇: 生成纯净静音轨。用于影视序列占位或精确控制分镜剪辑节奏。

---

### 5. 文本资产工具组 (TXT Nodes) 📄
专为分镜脚本、Prompt 序列及大模型上下文管理设计的工具。

*   **TrucyTxtBatchLoader (文本批量加载器)**:
    *   **排序模式**: 支持 `Alphabetical (A-Z)` 字母排序和 `Creation Time` 时间排序。
    *   **索引读取**: 通过 `index` 指定读取文件夹中特定的 TXT 文件，完美适配循环节点。
    *   **四路输出**: 
        1. `selected_content`: 选中文件的文本内容。
        2. `selected_filename`: 选中的文件名（不带后缀）。
        3. `merged_content`: 将文件夹内所有文本合并（中间空行隔开）。
        4. `merged_with_headers`: 将文件名作为标题，合并所有文本（适用于喂给 LLM 理解全剧）。
    *   **实时刷新**: 具备 `IS_CHANGED` 监测，外部修改文件后，下一次运行自动同步最新内容。

*   **TrucyTxtPreviewAndSave (预览与智能保存)**:
    *   **实时预览**: 直接在节点 UI 上显示当前文字内容。
    *   **智能保存**: 
        - 路径与文件名分离输入，支持带引号的 Windows 路径。
        - **自动编号**: 发现同名文件时自动递增 `_1`, `_2` 序号，防止覆盖历史分镜。
        - **强力容错**: 若填写的盘符不存在，会自动重定向至系统的“文档”文件夹下保存，确保流程不中断。
    *   **编码切换**: 可选 `UTF-8` 或 `ANSI (GBK)`，解决不同后期软件的乱码问题。

### 6. Audio Loader by Index (智能音频加载器) 🎵
摒弃了传统加载器一次性读取整个文件夹导致的内存浪费，专为影视音效流设计。

*   **极致性能**: 仅按 `index` 索引读取目标文件，不论文件夹里有几千首歌，加载都是瞬间完成。
*   **格式全通吃**: 自动扫描 `.mp3`, `.wav`, `.flac`, `.ogg`, `.m4a` 等多种格式。
*   **自动重采样**: 内置高质量重采样引擎（44.1k, 48k 等可选），确保输出的音频与视频合成节点的采样率要求完全一致。
*   **安全兜底**: 如果 `index` 超出文件总数或路径失效，节点会自动生成一段 **1秒钟 1000Hz 的标准提示音 (Beep)**。当你听到“哔”的一声，就知道素材位置该调整了，而不是程序报错挂掉。
